ETL Automation Mini Project

Python-based data ingestion, transformation, and reporting workflow

Project Overview

This mini project demonstrates an automated ETL (Extract, Transform, Load) pipeline built using Python and Pandas. It reads multiple CSV files from a folder, merges them into a unified dataset, performs data cleaning and transformation, and finally saves a clean CSV output ready for reporting or dashboarding.

  • Extract: Collects multiple sales data files from a folder using the glob module.
  • Transform: Merges datasets, calculates new columns such as Profit = Sales - Cost, and handles missing or duplicate values.
  • Load: Exports the final transformed dataset into a single clean CSV file.

Technical Implementation

The workflow was created in Jupyter Notebook and automated with Python scripting. The code uses the following modules:

  • Pandas: For reading, cleaning, and merging data.
  • OS and Glob: For handling multiple files within a specified directory.
  • Matplotlib: For visualizing aggregated sales and profit using bar charts.

This pipeline forms the foundation for automating repetitive data preparation tasks, reducing manual work and ensuring data consistency.

Output and Insights

The final output includes:

  • A cleaned and merged CSV file (clean_sales.csv).
  • Profit column derived from sales and cost data.
  • A bar chart visualizing Sales vs Profit by Product or Category.

This ETL mini project demonstrates automation and serves as a base for more complex workflows involving database connections, scheduling, and integration with BI tools like Power BI or Airflow.
